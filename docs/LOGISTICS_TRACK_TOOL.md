# 快递物流查询工具

## 交付信息

- 工具名：`logistics_track`
- 工具作用：根据快递单号查询物流状态、最新位置与近期物流节点。
- 支持参数：`tracking_no`、`carrier`、`phone_last4`。
- 触发场景：用户询问快递到了哪里、是否签收、正在派送还是预计何时送达。
- 返回类型：微信文本回复。
- 异常处理：单号缺失或格式错误、快递公司不支持、手机号后四位格式错误、物流服务未配置、服务限流或暂时不可用、未查询到物流信息。
- 测试用例：工具参数与格式化回复、Service 参数校验和轨迹排序、模拟快递100 HTTP 请求、凭证缺失处理。

## 能力边界

- 只查询即时物流轨迹，不创建物流变化订阅或主动提醒。
- 不获取取件码，不读取短信、菜鸟、丰巢或其他第三方账户数据。
- 单号只在当次 API 请求中使用；工具返回和日志均不展示完整单号。
- 部分快递公司可能要求收件人或寄件人手机号后四位。仅在用户主动提供时传给物流服务，不保存该数据。

## 环境配置

在本地 `.env` 中配置快递100正式账号的凭证，不要提交到仓库：

```properties
KUAIDI100_CUSTOMER=
KUAIDI100_KEY=
KUAIDI100_BASE_URL=https://poll.kuaidi100.com
```

未配置凭证时，项目仍能启动；调用工具会返回“物流服务尚未配置”的说明。`KUAIDI100_CUSTOMER` 和 `KUAIDI100_KEY` 可在快递100企业接口后台申请，不能使用普通网页查询页面的登录密码替代。

## 参数说明

| 参数 | 必填 | 说明 | 示例 |
|---|---|---|---|
| `tracking_no` | 是 | 快递单号，只允许字母、数字和连字符，长度 6 到 40 位 | `SF1234567890` |
| `carrier` | 否 | 快递公司，省略时交由物流服务自动识别 | `sf` |
| `phone_last4` | 否 | 收件人或寄件人手机号后四位 | `5678` |

支持的 `carrier` 值：`auto`、`sf`、`jd`、`ems`、`zto`、`yto`、`sto`、`yunda`、`deppon`、`jitu`。

## 示例用户输入

```text
帮我查一下 SF1234567890 到哪里了
```

```text
查询顺丰单号 SF1234567890，手机号后四位是 5678
```

## 示例 Function Calling JSON

```json
{
  "name": "logistics_track",
  "arguments": {
    "tracking_no": "SF1234567890",
    "carrier": "sf",
    "phone_last4": "5678"
  }
}
```
